### 관계형 데이터베이스 관리 시스템
#### 데이터
##### 데이터의 종류
- 구조적 데이터 : 테이블 형태로 정리된 데이터
- 반구조적 데이터 : XML,JSON
- 비구조적 데이터 : 미리 정의된 데이터 모델이 없거나, 미리 정의된 방식으로 정리되지 않은 정

##### 데이터 모델 개념
- 현실 서계의 정보를 컴퓨터에 표현하기 위해 단순화, 추상화아여 체계적으로 표현하는 방법
- 데이터, 데이터의 관계, 데이터의 의미 및 일관성, 제약조건 등 기술하기 위한 개념적 도구들의 모임
- 현실 세계를 데이터베이스에 표현하는 중간과정
- 데이터의 구조를 논리적으로 묘사하기 위해 사용함.

##### 분류
- 개념적 데이터 모델
- 논리적 데이터 모

##### 목적
- 클라이언트의 머릿속에 있는 개념, 시스템, 비지니스를 표현하는 데 도움이 되기 위해서
- 의사소통, 분류, 설명, 지정, 조사, 전개, 분석, 모방

##### 엔티티 관계 모델 (E-R 모델)
- ERD
- 개념적 데이터 모델링 생성 (업무 사양, 진술, 양식, 사규 등 -> ERD)
- 개념적 데이터 모델링 활용 (ERD -> 업무 이해 -> SQL 작성)
###### 이점
- 조직의 정보를 명확한 형식으로 문서화
- 정보 요구 사항의 범위를 쉽게 파악
- 쉽게 이해 가능한 데이터베이스 설계용 그림 지도 제공
- 여러 응용프로그램 통합을 위한 효과적인 프레임워크 제

##### 엔티티 관계 모델 구성 요소
- 엔티티
  - 중요한 어떠한 것
  - 실세계에 독립적으로 존재하는 유형, 무형의 정보
  - 서로 연관된 몇개의 속성으로 구성
- 속성
  - 엔티티를 설명, 분류, 기술하는 것
  - 데이터의 가장 작은 논리적 단위
- 관계
  - 엔티티 간의 관계
  - 선택가능성 (0 또는 1), 정도 (1 또는 M)
  - 카디널리티

##### 엔티티 관계 모델의 관계형 데이터베이스 구현
- 테이블의 각 데이터 행은 Primary Key를 통해 고유하게 식별됨.
- Foreign Key를 사용하여 두 데이터를 논리적으로 연관시킴
- 전체 데이터를 구하려면 조인 연산을 수행함.

#### 관계형 모델의 구성 요소
- 관계
   - 데이터 저장 요소
   - 관심 있는 데이터의 집합을 정의한 것
   - 테이블
- 연산자
   - 관계로부터 다른 관계 생성
   - 쿼리, 조인, 프로젝션, 합집합, 차집합 등
- 데이터 무결성
   - 데이터의 정확성 및 일관성 보장
   - 제약 조건
   - 관계형 모델의 주요 특징
- 정의
   - 관계 또는 2차원 테이블의 컬렉션
- 테이블
   - 단일 행 - 튜플
   - 칼럼 - 속성 (제약 조건 : PK 컬럼, FK 컬럼, NOT NULL 컬럼)
   - 필드 - 값 (NULL 값 : 값이 없음)
